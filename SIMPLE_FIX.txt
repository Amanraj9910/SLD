THE PROBLEM & THE FIX (Simple Explanation)
===========================================

WHAT HAPPENED:
==============

Your local machine has these files:
  ✓ app.py
  ✓ requirements.txt
  ✓ deploy.sh
  ✓ .deployment
  etc.

But GitHub repo doesn't have them yet (or Azure can't find them).

So when Azure tries to run: "bash deploy.sh"
It says: "bash: deploy.sh: No such file or directory"

Because the file doesn't exist in /home/site/wwwroot/ on Azure.


THE FIX:
========

Step 1: Make sure files are in GitHub
   Option A: Go to https://github.com/Amanraj9910/SLD
             Click "Upload files" and upload app.py, requirements.txt, etc.
   
   Option B: Use Git command: git add . && git commit -m "files" && git push
   
   Option C: Use GitHub Desktop and push

Step 2: Change startup command in Azure
   Azure Portal → App Service → Configuration
   
   Startup Command: python app.py
   
   (This tells Azure to run Python directly instead of looking for deploy.sh)

Step 3: Restart App Service
   Click Restart button

Step 4: Wait 2-3 minutes and check logs


WHY THIS WORKS:
===============

Before (broken):
  Azure looks for deploy.sh → Not found → Error

After (working):
  Azure runs: python app.py
  → app.py imports FastAPI from main.py
  → FastAPI backend starts
  → Serves frontend and API
  → Health check works ✓
