DEPLOYMENT INSTRUCTIONS FOR APP SERVICE

SOLUTION: BOTH FRONTEND & BACKEND IN SINGLE APP SERVICE

The Python backend now serves the React frontend as static files.
- Frontend: Pre-built at web_app/core/frontend/build/
- Backend: FastAPI on Python 3.11
- All API calls go to /api/* endpoints
- All other routes fallback to frontend (SPA routing)

PROBLEM FIXED:
- App Service was running PM2 static server instead of FastAPI backend
- Health check endpoint was inaccessible
- Now properly configured to run Python FastAPI backend with frontend

FILES UPDATED:
✅ app.py - Entry point that imports FastAPI and serves frontend
✅ main.py - Added frontend build serving with SPA fallback
✅ deploy.sh - Builds frontend with Node.js, then starts backend
✅ requirements.txt - All Python dependencies (consolidated)

DEPLOYMENT STEPS:

1. BUILD FRONTEND LOCALLY (if not already built):
   cd web_app/core/frontend
   npm run build
   cd ../../../

2. COMMIT & PUSH:
   git add .
   git commit -m "Configure App Service for FastAPI + Frontend"
   git push

3. IN AZURE PORTAL:
   a) Go to App Service → Configuration → General Settings
   b) Set Runtime Stack: Python 3.11
   c) Set Startup Command: bash deploy.sh
   d) Save changes

4. RESTART APP SERVICE:
   - Click Restart button
   - Monitor Logs → Application Logs
   - Check Log stream for startup messages

TROUBLESHOOTING:

If still showing PM2:
1. Check Runtime Stack = Python 3.11 (NOT Node.js)
2. Check Startup Command = bash deploy.sh
3. Restart App Service

If Frontend Not Loading:
1. Ensure build/ folder exists with index.html
2. Check logs for frontend build errors
3. If Node.js not available, frontend won't rebuild (but existing build should work)

If 502 Bad Gateway:
1. Check Application Insights
2. Verify all dependencies installed
3. Check Kudu console: https://YOUR-APP.scm.azurewebsites.net

If Port Error:
1. App Service sets PORT environment variable automatically
2. Python app reads from PORT env var
3. If issues, set in Portal → Configuration → Application Settings → PORT=8000

OPTIONAL: SEPARATE DEPLOYMENTS

If you prefer separate frontend/backend:

Option A: Azure Static Web Apps for Frontend
   - Deploy web_app/core/frontend/build to Static Web Apps
   - Configure API backend proxy to App Service
   - More scalable but more complex

Option B: Separate Node.js App Service for Frontend
   - Deploy frontend to separate App Service (Node.js runtime)
   - Keep backend on this App Service (Python runtime)
   - More resources but better isolation
