QUICK FIX - Files Not Deployed
================================

PROBLEM: deploy.sh file exists locally but wasn't pushed to GitHub/Azure.

SOLUTION OPTIONS:

OPTION 1: Use GitHub Web Interface (Easiest)
============================================

1. Go to: https://github.com/Amanraj9910/SLD

2. Navigate to root folder

3. Click "Add file" → "Upload files"

4. Upload these files from your local machine:
   ✓ app.py
   ✓ .deployment
   ✓ requirements.txt
   ✓ deploy.sh
   ✓ app_startup.py
   ✓ oryx-manifest.toml

5. Add commit message: "Add deployment config files"
   Click "Commit changes"

6. Go back to Azure Portal and Restart App Service


OPTION 2: Use Git Command Line (If You Have Git)
================================================

1. Install Git from: https://git-scm.com/download/win

2. Open PowerShell and run:
   cd c:\Users\admin\Downloads\Single-Line-Diagram\SLD
   git add .
   git commit -m "Add deployment configuration files"
   git push

3. Go to Azure Portal and Restart App Service


OPTION 3: Use VS Code Git Integration
=====================================

1. Open VS Code
2. Open folder: c:\Users\admin\Downloads\Single-Line-Diagram\SLD
3. Click Source Control (left sidebar)
4. Stage all changes: Click +
5. Enter message: "Add deployment configuration files"
6. Click Commit
7. Click ... → Push

8. Go to Azure Portal and Restart App Service


OPTION 4: Update Startup Command in Azure Portal (Quickest)
===========================================================

1. Go to Azure Portal → App Service → Configuration

2. General Settings → Startup Command

3. Change from: bash deploy.sh
   To: python app.py

4. Click Save and Restart

   This will run app.py directly, which handles all setup


RECOMMENDED: OPTION 4
====================

Fastest way forward:

1. In Azure Portal:
   Configuration → General Settings
   
   Startup Command: python app.py
